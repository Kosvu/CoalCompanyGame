# CoalCompanyGame

Небольшая игра-кликер про угольную компанию, в которую играют... HTTP-запросами. Я написал её, когда изучал горутины и каналы, — хотелось попробовать конкурентность на чём-то живом, а не на учебных примерах.

## Суть игры

Вы управляете угольной компанией. Нанимаете шахтёров — каждый шахтёр работает в **своей горутине** и через канал присылает добытый уголь, пока у него не кончится энергия. Уголь превращается в деньги на балансе. Цель — заработать на всё оборудование шахты: кирку (3 000), вентиляцию (15 000) и вагонетку (50 000). Купили всё — можно завершать игру и смотреть результат: итоговую статистику и время прохождения.

Шахтёры бывают трёх типов:

| Тип | Цена | Добыча | Скорость | Энергия |
|---|---|---|---|---|
| small | 5 | 1 уголь | раз в 3 сек | 30 |
| normal | 50 | 3 угля | раз в 2 сек | 45 |
| strong | 450 | 10 угля | раз в 1 сек | 60 |

Ещё капает пассивный доход, чтобы можно было нанять первого шахтёра.

## Что под капотом

Для меня это в первую очередь упражнение на конкурентность:

- Каждый шахтёр — отдельная горутина, отдающая уголь через канал (`Run(ctx) <-chan Coal`)
- Баланс компании защищён мьютексом — уголь приходит из многих горутин одновременно
- Остановка всех шахтёров при завершении игры сделана через отмену контекста
- Типы шахтёров реализуют общий интерфейс `Miner`

Стек: Go, gorilla/mux, стандартная библиотека.

## API

| Метод | Путь | Что делает |
|---|---|---|
| GET | `/company` | Состояние компании: баланс, шахтёры, оборудование |
| POST | `/miners` | Нанять шахтёра (тип в теле запроса) |
| GET | `/miners` | Список всех шахтёров |
| GET | `/miners/{class}` | Шахтёры конкретного типа |
| GET | `/equipment` | Список оборудования и что уже куплено |
| POST | `/equipment/{name}` | Купить оборудование |
| POST | `/company` | Завершить игру (если всё оборудование куплено) |

## Как поиграть

```bash
go run .
```

Сервер поднимется на `:9091`. Дальше — из curl или Postman:

```bash
# посмотреть состояние компании
curl localhost:9091/company

# нанять шахтёра
curl -X POST localhost:9091/miners -d '{"class": "small"}'

# купить кирку, когда накопите
curl -X POST localhost:9091/equipment/Pickaxe

# завершить игру
curl -X POST localhost:9091/company
```

## Чему я научился

- Горутины, каналы, select и отмена через context — на практике, а не по учебнику
- Синхронизация доступа к общему состоянию через мьютекс
- Интерфейсы как способ описать разные типы с общим поведением
